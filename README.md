## ☁️ DevOps & Cloud

### Azure DevOps (how I usually structure delivery)
- **Pipeline flow**: `build → test → lint → SAST/dep scan → package → deploy`
- **Release controls**: approvals, gates, environment-specific variables
- **Versioning**: semantic tags, immutable artifacts, predictable rollbacks
- **Secrets**: Key Vault / secret stores, minimal blast radius, rotation
- **IaC**: plan/apply workflows, drift awareness, environment parity

### Operational principles
- **Reproducibility > hero debugging**
- **Small batch deployments** (safe, frequent)
- **Observability is a feature** (not “later”)
- **Security controls live in the pipeline** (not in a slide deck)

## 🧩 Full-Stack Engineering

### What I optimize for
- **Clean contracts**: OpenAPI schemas, typed DTOs, validation
- **Secure defaults**: authz rules, rate limiting, safe headers, input handling
- **Performance**: caching, indexes, async work, profiling
- **Reliability**: timeouts, retries, idempotency, sane failure modes
- **Maintainability**: tests, linting, clear architecture and docs
